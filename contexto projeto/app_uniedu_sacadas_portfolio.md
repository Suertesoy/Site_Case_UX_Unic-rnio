# APP UNIEDU — Sacadas de Portfólio e Product Thinking

## Visão Geral do Projeto

APP UNIEDU é um protótipo mobile-first de plataforma de ensino gamificada voltada para estudantes de UX/UI Design.

O projeto evoluiu de um conceito mais lúdico para uma experiência premium, adulta e focada em retenção, motivação comportamental e construção de carreira.

O grande diferencial do case está em mostrar:
• profundidade de Product Thinking
• preocupação com retenção
• refinamento visual progressivo
• decisões orientadas por comportamento
• integração entre UX + UI + implementação real
• pensamento sistêmico entre design e desenvolvimento

---

# 1. SACADAS DE PRODUTO E UX

## Video-First Learning

### Problema identificado
Os “cases” estavam competindo visualmente com o conteúdo principal do produto.

### Decisão
Reposicionar o aplicativo como:
“uma plataforma de aprendizagem contínua em vídeo”.

### Implementação
• O card “Você está aqui” virou o Hero principal da Home
• O checklist onboarding passou a ser secundário
• Os cases foram tratados como marcos/consequências da evolução

### Impacto de UX
• clareza de propósito
• redução de distração
• aumento de foco no conteúdo principal
• melhora do modelo mental do usuário

---

## Checklist de Exploração do Ecossistema

### Problema
Usuários poderiam entrar apenas nas aulas e ignorar funcionalidades importantes do produto.

### Decisão
Criar uma trilha inicial temporária incentivando:
• comentar
• ganhar conquista
• publicar
• resgatar recompensa

### Sacada forte de portfólio
O checklist não foi pensado como gamificação eterna.

Ele funciona como:
“onboarding comportamental”.

Depois que o usuário aprende o ecossistema:
o componente desaparece.

### Impacto
• descoberta de funcionalidades
• ativação inicial
• educação do usuário
• redução de abandono de features

---

## Gamificação Voltada para Carreira

### Problema
Gamificação tradicional infantiliza produtos educacionais.

### Decisão
Transformar a lógica de recompensa em:
• progresso profissional
• construção de reputação
• consistência de estudos
• visibilidade de evolução

### Exemplos
• ranking
• streaks
• pontos
• desafios
• badges
• recompensas reais
• networking

### Importante
O projeto deliberadamente manteve:
“Pontos”
e não “Pontos de Especialização”.

Porque:
• é mais natural
• menos corporativo
• mais próximo de plataformas reais

---

## Adultificação da Marca

### Problema
O conceito de unicórnio atraía estética infantil.

### Estratégia
Manter o DNA da marca sem infantilizar o produto.

### Soluções
• dark mode premium
• glassmorphism
• unicórnio neon/glass
• redução de saturação
• tons mais profundos
• interface mais “tech”

### Insight forte
A solução não foi:
“remover o unicórnio”.

Foi:
“reposicionar visualmente o símbolo”.

---

## Dark Mode como Feature de Demonstração

### Sacada
O Dark Mode deixou de ser configuração escondida e virou parte da experiência.

### Implementações
• toggle rápido no Login
• toggle rápido na Home
• persistência de tema
• adaptação automática de assets

### Impacto
Excelente para:
• apresentações
• vídeos de portfólio
• demonstração visual
• percepção premium

---

## Reset de Protótipo

### Problema
Fluxos onboarding/login só aconteciam uma vez.

### Decisão
Adicionar:
“Resetar Protótipo”.

### Impacto
• facilidade para testes
• facilidade para gravações
• demonstração repetível
• melhor usabilidade para recrutadores

### Sacada de produto
A funcionalidade foi criada pensando:
não apenas no usuário final,
mas no contexto de apresentação do case.

---

## Contextualização Inteligente da Comunidade

### Problema
A comunidade herdava automaticamente o contexto da última aula acessada.

### Decisão
Separar:
• publicação livre
• publicação contextual

### Impacto
• redução de fricção
• prevenção de comportamento inesperado
• mais controle para o usuário

### Insight
Isso demonstra preocupação com:
estado global
persistência de contexto
e previsibilidade de interface.

---

## Navegação Mais Nativa

### Problema
Usuário precisava usar o menu inferior para sair do ranking.

### Decisão
Adicionar navegação contextual com seta de retorno.

### Impacto
• sensação mais nativa
• redução de esforço cognitivo
• melhor fluxo mobile

# 2. SACADAS DE UI DESIGN

## Sistema de Cores Premium

### Problema
As cores vibrantes funcionavam no Light Mode mas cansavam no Dark Mode.

### Solução
Criar paleta específica para Dark Mode:

Background:
#120E10

Surface:
#1E1A1D

Pink Primary:
#D81B60

Pink Accent:
#F06292

### Sacada
Não foi apenas:
“inverter cores”.

Foi criado:
um sistema visual próprio para ambientes escuros.

---

## Jewel Tones no Dark Mode

### Problema

### Solução

• vinho
• índigo
• teal escuro
• roxo profundo

### Impacto
• aparência premium
• menos fadiga visual
• maior sofisticação
